# KOKI Portfolio — 要件定義書

> 作成: 2026-06-27 / 最終更新: 2026-06-27

---

## 1. プロジェクトの本質

**KOKIというクリエイティブエンジニアの存在を、開いた瞬間に体感させる。**

mantis.worksのように「人体 + ドラマチックな3D空間 + タイポグラフィ」で圧倒する。
宇宙・地球・人間の3つのスケールを並べることで、誰にもできないビジュアルを作る。

---

## 2. ターゲット・ゴール

**就活との兼用は捨てる。学校展示に一本化。**

| 項目 | 内容 |
|------|------|
| 対象①（授業発表） | 教員・同級生（プロジェクター or 画面共有） |
| 対象②（オープンキャンパス） | 高校生・保護者・教員（研究室展示、通りすがり） |
| ゴール | **「大学2年がこれを一人で作った」という事実でぶん殴る** |
| 将来 | フルスタック寄りWebエンジニア志望。就活ポートフォリオは別途考える |

### 展示シーン別の注意点

**授業発表（プロジェクター）**
- KOKIが操作しながら実況する想定
- ローディングが長いと空気が死ぬ → 高速ロードが必要
- 各フェーズで「ここはこうやって作った」と言える余地を残す

**オープンキャンパス（研究室展示）**
- 通りすがりの人が足を止める必要がある → 開いた瞬間が全て
- 自分で操作してもらう or KOKIが実演
- QRコードでスマホから体験できると嬉しい
- 技術説明の看板 or 隣にスライドがあれば、サイトは体験に徹していい

## 3. ポジショニング

「宇宙空間に存在するKOKI」というビジュアルで、クリエイティブエンジニアとしての存在感を出す。

参照: mantis.works（ワイヤーフレーム人体 × タイポグラフィ × 暗背景）
差別化: 地球・宇宙という圧倒的なスケール感を背景に持つ

---

## 4. コンセプト ✅ 確定

### ストーリー：「作る人間の宣言」

> Webを作る人間がいる。その手で、世界に触れようとしている。

**ビジュアル**: ワイヤーフレームの手が宇宙空間で地球を掴んでいる  
**意味**: 手 = コードを書く手・作る手 / 地球 = Web・インターネット  
**印象**: 説明不要。開いた瞬間に「こいつ只者じゃない」と思わせる

### 構成

```
[HERO] 宇宙空間 + 地球 + ワイヤーフレームの手（両サイドから掴む）
       タイポ: "KOKI" + "Web Engineer / Creative Developer"
       ↓ scroll
[PORTFOLIO] Skills / Works / Contact
```

### 参照とKOKIの差別化

| mantis.works | KOKI |
|--------------|------|
| ワイヤーフレーム人体が立っている | ワイヤーフレームの手が地球を掴んでいる |
| グレー背景 | 宇宙・暗黒背景 |
| 人間スケール | 神スケール（手 > 地球）|

ワイヤーフレームの質感だけ参照、構図・スケール・世界観は完全にオリジナル。

## 5. 体験フロー（確定版）

```
[HERO] 宇宙空間 + 地球 + ワイヤーフレームの手
  - ロード完了と同時に全画面で出現
  - 手が地球を掴んでいる構図
  - タイポ: "KOKI" + キャッチコピー
  - スクロール促すインジケーター
    ↓ scroll
[PORTFOLIO]
  - Skills（技術スタック）
  - Works（このサイト自体 + あれば他作品）
  - Contact（GitHub / SNS / メール）
```

---

## 6. 必要なコンテンツ（KOKIが用意するもの）

### 6.1 About
- 自己紹介文（2〜3行）
- 現在地（学生 / 社会人 / 地域など）

### 6.2 Skills
- 技術スタック（習熟度付き）
  - 例: Three.js ★★★★☆、Next.js ★★★★★、TypeScript ★★★☆☆...
- 得意なこと一言（例: 「モーションとコードの境界をなくす」）

### 6.3 Works（任意・このサイト以外）
- 他に見せたい作品があれば追加
- なければ「このサイト」の技術解説カードでも成立する

### 6.4 Contact
- メールアドレス or お問い合わせフォーム
- GitHub URL
- SNS（X/Twitter など）

## 7. 技術要件・優先順位

### Priority 1（展示に必須）
- [ ] 詩的テキスト演出（落下中にフェードイン/アウト）
- [ ] ローディング画面（待機中も絵になる、長くても死なない）
- [ ] 体験の流れが途切れなく動く（スタッター・落ちなし）
- [ ] PC（プロジェクター想定）で60fps安定

### Priority 2（体験クオリティ）
- [ ] サウンドデザイン（宇宙の静寂 → 大気圏の轟音 → 海の音）
- [ ] FPSハンド or 視点強化
- [ ] フレネル反射・水面強化
- [ ] 体験エンドの演出（海面で止まる・余韻）

### Priority 3（展示環境対応）
- [ ] スマホ対応（QRコードでオープンキャンパス来場者が体験）
- [ ] パフォーマンス最適化（展示PCのスペックが低い可能性）
- [ ] キオスクモード検討（自動ループ再生）

## 9. 「完成」の定義

以下がすべて満たされたら展示可能とする:

1. ✅ 宇宙→海面の体験が途切れなく動く
2. ✅ 開いた3秒以内に「すごい」と思わせるビジュアルが出る
3. ✅ 詩的テキストが体験に乗っている
4. ✅ PC（プロジェクター環境）で60fps安定
5. ✅ ローディングが耐えられる（20秒以内 or 演出で誤魔化せる）
